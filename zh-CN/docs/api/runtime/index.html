运行时 API

运行时提供确定性组合、取消、配置、时间域控制和 Phaser 适配契约。

FeelPlayer​

创建不可变时间线。条目支持绝对 at 时间、通过 hold: false 并行播放、确定性概率、有限循环、方向、生命周期观察和子播放器。

FeelHandle​

表示一次可等待的运行。可读取 state、等待 completion，或调用幂等的 pause()、resume()、stop()、skipToEnd() 和 setIntensity()。

Scheduler​

调度时间和随机源均可注入。固定时钟、seed、配置和上下文无需启动 Phaser 或浏览器即可产生相同顺序。

TimeDomain​

TimeDomain 暴露不可变的倍率、暂停和 revision 状态。租约会相乘倍率并合并暂停请求；time.freeze、time.scale 和 time.hitStop 通过真实时间释放各自租约，不会干扰重叠效果。

FeedbackRegistry​

FeedbackRegistry 在播放前拒绝未知类型和字段，并从 JSON 安全配置创建播放器。TargetResolverRegistry 可注册并注销命名 resolver，且无需序列化函数。

RuntimeAdapter​

RuntimeAdapter 负责 Phaser 特定的 Camera、事件、目标查找、数值 Tween 和 TimeDomain 同步。PhaserRuntimeAdapter.createTimeDomains(scene) 在倍率、暂停、shutdown 和恢复期间保留 Scene Clock 与 Tween 各自的原始状态。

Feedback Schema DSL​

defineFeedbackSchema(type, shape) 校验未知字段、标量类型和数值边界，提供默认值，并输出 JSON Schema。

配置示例​

const registry = createDefaultFeedbackRegistry();
const player = registry.createPlayer({
  id: 'impact',
  feedbacks: [
    {type: 'event.emit', params: {event: 'impact'}, hold: false},
    {type: 'camera.shake', params: {duration: 120}, at: 20, hold: false},
  ],
});

player.on(event => console.log(event.type, event.time));
const handle = player.play({target, runtime}, {seed: 42});
handle.pause();
handle.resume();
await handle.completion;
